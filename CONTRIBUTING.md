# Contributing

1. Create a focused branch from `main`.
2. Keep canonical language behavior separate from fictional speech aliases.
3. Add tests for parser, compiler, runtime, and web-profile changes.
4. Run `npm ci`, `npm run ci`, and `npm run playground` for browser-facing changes.
5. Update `SPEC.md`, relevant docs, and the changelog when behavior changes.
6. Open a pull request with the problem, implementation, tests, compatibility impact, and security considerations.

All satire must remain clearly fictional and must not be presented as an authentic quotation or endorsement.
